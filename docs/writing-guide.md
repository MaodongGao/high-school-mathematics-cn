# 写作与录入约定

## 一次整理的最小单元

1. 从 `sources/catalog.csv` 选择或登记一份原始 PDF。
2. 只查看一小段连续页面，先判断主题边界。
3. 将内容放入对应章节；主题不匹配时再新建章节。
4. 用 `\sourcetodo{...}` 记录尚未核对的页码或疑问。
5. 编译并检查公式、图、分页与交叉引用。
6. 逐页核对完成后更新来源状态。

## 文件命名

- 章节使用两位序号和英文短名，例如 `02-functions.tex`。
- 图片使用小写英文和连字符，例如 `projectile-trajectory.pdf`。
- `\label` 加学科语义前缀，例如 `eq:physics-projectile-x`。

## 内容结构

- 每章以学习目标开始。
- 首次出现的概念放入 `definitionbox`。
- 完整解题过程放入 `examplebox`。
- 每个例题标题必须以“原稿例：”“原稿型：”“原稿方法：”或“补充：”标明来源性质；整理者新增的题只能标为“补充：”。
- 新增题应具有代表性、启发性和迁移价值，不以解法短或计算快为选择标准。
- `lawbox`、`corollarybox`、`examplebox` 分别使用蓝、绿、金三种可区分的视觉层级。
- 原稿旁注或待补内容放入 `notebox` 或 `\sourcetodo`。
- 每个主要知识点后安排少量 `exercisebox` 练习。

二级结论要根据逻辑作用安排位置：解题前必需的工具放在例题前；由例题归纳出的结论放在例题后。不要先完整给出结果，再用例题重复推导。

“高信息密度”只用于描述项目目标，不写进教材正文。它表示压缩重复叙述，同时保留理解链条、条件、图形和有启发性的完整解法；不得改写成“最短解”“速查”等速度导向表达。

## 公式与单位

- 多行推导使用 `align`，不要用连续的手工换行。
- 向量使用 `\vec{v}`，微分使用 `\dd x`。
- 数集、向量和微分分别使用项目定义的 `\N`、`\vec{v}` 与 `\dd x` 等命令，以保持符号统一。
- 对公式、图和表使用 `\label` 与 `\cref`，避免手写编号。

## 图形

优先用 TikZ/PGFPlots 重画几何图与函数图。扫描裁图只作为过渡，并在同一章节留下替换说明。

几何例题必须配有与解法一致的 TikZ 图；重要的空间二级结论在仅靠文字难以迅速重建时也应配图。图中应画出解答实际使用的辅助线、射影、截面或角，而不是只画一个装饰性几何体。
